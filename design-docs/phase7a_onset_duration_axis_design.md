# phase7a_onset_duration_axis_design.md
## 主訴別DDxナビ：発症経過（急性・亜急性・慢性・反復・不明）入力軸の設計案

対象バージョン：ddx-8.0-generic-beta.html（本ドキュメント作成時点でHTMLは変更していない）

本ドキュメントはPhase 7a（設計検討のみ）の成果物であり、実装そのものではない。
記載した実装方針は「もし実装するなら」の設計案であり、本フェーズでの実装可否はsummaryで判定する。

## 1. 現状データの確認（設計の前提）

実装方針を検討するにあたり、既存のRAW_FINDINGS（473件）・FINDINGS（31件）に
発症経過（急性/慢性等）を示す項目がどの程度既に存在するかを確認した。

- label に「急性」を含むもの：**2件**（`pe_hindlimb_paresis`＝急性後肢麻痺、`hx_acute_blindness`＝急性の失明）
- label に「慢性」を含むもの：**1件**（`hx_chronic_cough`＝慢性咳（2ヶ月以上））
- label に「亜急性」を含むもの：**0件**
- label に「反復」を含むもの：**1件**（`hx_repeated_vomiting`＝反復する嘔吐）
- 期間の閾値を明示するもの：**2件**（`hx_anorexia_persistent`＝3日以上の食欲不振、`hx_chronic_cough`＝慢性咳2ヶ月以上）

**確認結果**：発症経過を扱うRaw所見は現状きわめて少数（504件中5件程度）であり、対象となる21主訴（嘔吐・下痢・咳・跛行・神経症状等）のほとんどにおいて、急性/慢性を直接示すRaw所見が用意されていない。したがって「レベル2：既存Raw所見にマッピング」は、咳（`hx_chronic_cough`）等ごく一部の主訴でしか現状は機能しない。

また、上記5件のRaw IDがDERIVED_PROBLEMSのtriggerでどう使われているかも確認した。いずれも`any`条件（支持所見の一つ）として使われており、単独で候補を確定させる`all`条件の主要素にはなっていない（例外は`dp_feline_hepatic_lipidosis_typical`のように複数の`all`条件の一つとして使われるケースのみ）。これは、既存の診断ロジックが「発症経過を含む個別の所見を、候補を絞り込みすぎない補助的な支持材料として扱う」設計方針を既に採っていることを示しており、本設計案の安全方針（経過をフィルターにしない）はこの既存方針と整合する。

## 2. 急性/慢性軸が必要と判断される理由

主訴によっては、発症経過により初期対応・優先鑑別が大きく変わる。具体例は「7. 主訴別必要度」を参照。
特に以下のパターンで、経過情報がないことによる「候補の広がりすぎ」が生じ得る：

- 同一主訴内で、緊急性の高い急性疾患と、経過観察でよい慢性疾患が同じ重みで並列表示される
- 慢性疾患を示唆する所見（低Alb・腸管壁肥厚等）と、急性疾患を示唆する所見（電解質異常等）が同時に選択された際、どちらの文脈で読むべきかがUI上分かりにくい
- 咳・跛行・神経症状のように、急性/慢性で治療の緊急度・専門判断の要否が大きく変わる主訴では、経過情報がないと若手が「次に何を考えるか」の手がかりを失いやすい

ただし、これは「経過情報を追加すべき」という結論を直ちに支持するものではなく、以下の安全方針・実装レベルの制約のもとで、段階的に検討すべきものである。

## 3. 推奨実装レベルの比較

| レベル | 内容 | メリット | デメリット | Phase 7での実装可否 |
|---|---|---|---|---|
| **レベル1：UI入力のみ** | 経過を選べる。選択中所見エリア（または専用エリア）に「経過：急性」等を表示。診断候補計算には一切使わない。 | 最も安全。診断ロジックに一切触れないため医学的レビュー不要。field feedbackで「そもそも急性慢性選択が必要か」を検証できる。 | 候補順位・支持所見には反映されない（UIの表示のみ）。 | **可能**（UIの追加のみ） |
| **レベル2：既存Raw所見へのマッピング** | 「急性」を選んだら`hx_repeated_vomiting`等の既存急性/慢性系Raw所見と連動させる（例：選ぶと関連チェックボックスを操作可能にする、または関連所見を強調表示する）。 | 既存の診断ロジック（trigger）をそのまま活用できる。 | 上記1章の通り、対応するRaw所見が咳等ごく一部の主訴にしかなく、大半の主訴（嘔吐・下痢・跛行等）では連動先が存在しない。中途半端な適用は「一部の主訴だけ経過が効く」という不整合なUXを生むリスクがある。 | **条件付きで可能**（咳など対応Rawがある主訴に限定するなら可能。全主訴への一律適用は不可） |
| **レベル3：疾患カードへのonset/durationメタデータ付与＋候補順位への重み付け** | 252疾患カードそれぞれに`onset: ['acute']`等のメタデータを付与し、候補の表示順・支持所見計算に反映する。 | 候補の実用性が最も上がる（発症経過に応じた優先表示が可能になる）。 | 252疾患カード全件に対する医学的な妥当性監査が必要（「この疾患は急性/慢性どちらに分類されるべきか」という判断は医学的レビューそのものであり、UI実装の範囲を超える）。診断ロジック・candidate role・DERIVED_PROBLEMSに影響する可能性が高い。 | **不可**（Phase 8以降の医学的再監査対象） |

### 暫定推奨
**Phase 7aでは、レベル1（UI入力のみ、診断ロジックには一切使わない）を推奨する。**
レベル2は「咳」等ごく一部の主訴に限定してもなお、以下の理由で今回の実装対象からは除外する：
- 一部の主訴だけ経過選択が意味を持ち、大半の主訴では見た目だけの選択になり、若手にとって「効いているのか分からない」UIになる
- 咳の`hx_chronic_cough`は「2ヶ月以上」という具体的な期間定義を持つ既存の医学的判断であり、これをUIの経過選択（「慢性：2〜3週間以上」）と単純に連動させると、期間定義の不一致（2ヶ月以上 vs 2〜3週間以上）が生じ、かえって誤解を招く

レベル3はPhase 8以降の医学的再監査対象として明確に分離する。

## 4. UI配置案の比較

| 案 | 配置 | メリット | デメリット |
|---|---|---|---|
| **案A：主訴カード内** | 各主訴カードの「プロブレムリストで所見を選ぶ」ボタンの近くに経過選択を配置 | 主訴ごとに自然に選べる。若手が最初に考えやすい（嘔吐・下痢・咳・跛行等で直感的）。 | 22主訴すべてにUIが増える（同じ選択肢が22箇所に重複表示され冗長）。主訴カードを経由しない入り口（プロブレムリスト直接検索・所見カテゴリタブ）から入った場合に選択機会がない。 |
| **案B：プロブレムリスト検索パネル上部** | 犬猫フィルター・直接検索・選択中所見の近くに共通の経過選択を1箇所配置 | UIとして一箇所にまとまる。どの入り口（主訴カード／直接検索／タブ）から入っても共通して使える。選択中所見エリアと連動して管理しやすい。実装箇所が1つで済む。 | 主訴カードだけ見ている段階では経過を意識しにくい（プロブレムリストパネルを開くまで経過選択に気づかない）。 |

### 暫定推奨
**案B（プロブレムリスト検索パネル上部に共通の経過選択を1箇所配置）を第一候補とする。**
理由：
- 既存のRaw/FINDINGSを増やさずに実装しやすい（新規データ追加不要）
- 選択中所見エリアと連動しやすい（同じ「選択状態」の管理の枠組みに乗せられる）
- 22主訴すべてに共通して使え、UIの重複を避けられる
- Phase 7の軽微修正範囲（新規UIコンポーネント1つ＋表示連動のみ）に収まりやすい

将来的に「主訴を開いた時点で経過を意識してほしい」というfield feedbackが得られた場合は、案Aを補助的に追加する（主訴カードから案Bの経過選択欄へスクロール誘導する等）ことも検討可能だが、これは本設計案の対象外とする。

| 主訴 | 必要度 | 理由・想定される急性/慢性の分岐例 |
|---|---|---|
| 嘔吐 | **高** | 急性：急性胃腸炎、異物/閉塞、膵炎、中毒、DKA、AKI、アジソン病／慢性：慢性腸症、消化管型リンパ腫、CKD、甲状腺機能亢進症、肝胆道疾患、慢性膵炎 |
| 下痢 | **高** | 急性：急性腸炎、寄生虫、食事性、出血性下痢症候群、パルボ等／慢性：慢性腸症、PLE、リンパ腫、膵外分泌不全、内分泌疾患 |
| 食欲不振 | **中** | 急性の食欲不振は多くの急性疾患の随伴症状として非特異的。慢性（数日以上持続）は`hx_anorexia_persistent`が既に存在し、肝リピドーシス等の重要な手がかりになる。 |
| 体重減少 | **高** | 慢性経過が前提となることが多く（腫瘍、内分泌、慢性炎症）、急性の体重減少は稀。経過情報により「慢性消耗性疾患を考えるべきか」の分岐に有用。 |
| 咳 | **中〜高** | 急性：誤嚥性肺炎、肺水腫、気胸、胸水、気道閉塞／慢性：慢性気管支炎、気管虚脱、心疾患、肺腫瘍、フィラリア等。既存`hx_chronic_cough`（2ヶ月以上）があり、レベル2適用の候補になり得るが期間定義の整合性に注意が必要（3章参照）。 |
| 呼吸困難 | **中** | 急性発症が救急対応上重要（気胸・肺水腫等）。慢性咳嗽との組み合わせで心疾患・肺腫瘍等の可能性が変わる。 |
| 排尿異常 | **中** | 急性の尿道閉塞は救急疾患として経過に関わらず最優先表示すべき（安全方針参照）。経過情報は下部尿路疾患の再発性パターン（反復性）の把握に有用。 |
| 多飲多尿 | **中** | 急性発症は珍しく、慢性・進行性の経過が診断の手がかりになることが多い（CKD、糖尿病、副腎皮質機能亢進症等）。 |

| ふらつき/斜頸/眼振 | **高** | 急性前庭疾患（特発性・末梢性）と慢性進行性の中枢性疾患（腫瘍・炎症性CNS疾患）で対応・予後説明が大きく異なる。 |
| 跛行 | **高** | 急性：外傷、骨折、脱臼、靭帯損傷、感染性関節炎／慢性：変形性関節症、股関節形成不全、免疫介在性多発性関節炎、骨腫瘍、神経疾患。 |
| 皮膚掻痒 | **中** | 慢性・再発性の経過はアレルギー性疾患（アトピー等）を、急性発症は接触性・寄生虫性・アレルギー反応を示唆しやすい。 |
| 皮膚腫瘤 | **中〜高** | 急速増大（急性的変化）は肥満細胞腫等の悪性腫瘍を積極的に疑う根拠になる（Phase 5eの高リスク治療UIとも関連）。「反復性」は該当しにくく、急性/慢性というより「増大速度」の情報として扱う方が適切な可能性がある。 |
| 眼の異常 | **中** | 急性の視力低下・眼痛は眼科救急（緑内障・網膜剥離等）として即日対応が必要。慢性経過は白内障等の非緊急疾患を示唆しやすい。 |
| 鼻汁 | **低〜中** | 急性は感染性（ウイルス性上気道感染等）、慢性・片側性は腫瘍・真菌性鼻炎・歯周疾患由来等を示唆するが、経過よりも片側性/両側性・血性の有無等の他の所見の方が鑑別への寄与が大きい可能性がある。 |
| 流涎 | **低** | 多くは急性の悪心・口腔内異物・中毒等に伴う随伴症状であり、経過情報単独での鑑別への寄与は小さい。 |
| 腹囲膨満 | **中** | 急性（腹水の急速貯留、胃拡張捻転等）と慢性（腹腔内腫瘤、慢性肝疾患による腹水等）で緊急度が大きく異なる。 |
| 出血傾向 | **中** | 急性発症は中毒（殺鼠剤等）・DIC・急性免疫介在性血小板減少症を、慢性・反復性は遺伝性凝固異常等を示唆し得るが、出血傾向は経過よりも所見の重症度（部位・程度）が優先される場面が多い。 |
| 虚脱 | **低〜中** | 虚脱自体は急性事象であることがほとんどであり、経過選択の必要性は他の主訴に比べて低い。基礎疾患（心疾患等）の慢性経過を問う場面はあるが、主訴自体としては急性が前提となりやすい。 |
| 中毒/誤食 | **低** | 定義上、曝露は急性事象であり、経過選択の必要性は低い（曝露からの経過時間は別の情報として扱う方が適切）。 |
| 生殖器/乳腺の異常 | **中** | 産褥期等のタイミング情報（既存のhx_postpartum等）の方が経過（急性/慢性）よりも鑑別への寄与が大きい可能性がある。 |

**必要度が高い主訴（優先実装候補）**：嘔吐、下痢、体重減少、発作、ふらつき/斜頸/眼振、跛行
**必要度が中程度の主訴**：食欲不振、咳、呼吸困難、排尿異常、多飲多尿、皮膚掻痒、皮膚腫瘤、眼の異常、腹囲膨満、出血傾向、生殖器/乳腺の異常
**必要度が低い主訴**：鼻汁、流涎、虚脱、中毒/誤食

## 6. 表示文言案（確定案）

**見出し**：経過を選ぶ
**補足文**：急性・慢性で鑑別の優先度が変わります。不明な場合は選ばなくて構いません。
**選択肢**（ラジオボタン、単一選択）：
- 急性（数時間〜数日）
- 亜急性（数日〜2週間程度）
- 慢性（2〜3週間以上）
- 反復性（良くなったり悪くなったりを繰り返す）
- 不明

## 7. 安全設計（実装する場合に必ず守る方針）

- **critical mimic / danger / 救急候補は、経過に関わらず表示を維持する。** 例：尿道閉塞・気胸・アナフィラキシー等は、「慢性」を選んでいても除外・降格させない。
- **未選択は陰性ではなく「未評価」として扱う。** 経過を選ばなかった場合、急性・慢性いずれの候補にもペナルティを与えない（レベル1では診断計算に使わないため、この点は自動的に満たされる）。
- **「反復性」は「慢性」と同一視しない。** 反復性は「間欠的パターン」という独自の情報であり、慢性（持続的経過）とは意味が異なる。UI上も別の選択肢として提示する。
- **「不明」を必ず用意する。** 経過が分からない場合に無理に急性/慢性を選ばせない。
- **経過選択は解除可能にする。** 一度選んだ後、再度クリックする等で「未選択」の状態に戻せるようにする（ラジオボタンの仕様上、専用の解除操作＝同じ選択肢の再クリックまたは「解除」ボタンを設ける）。
- **選択中所見エリア、または専用の表示エリアに現在の経過選択を表示する。** 選択中所見エリアに「経過：急性」等のチップを表示するか、隣接する専用エリアに表示するかは実装時に検討する（レベル1ではいずれも診断ロジックに影響しないため、視認性を優先して選ぶ）。
- **plResetで経過選択も解除する（暫定推奨）。** 「選択中の所見」を含む入力全体をリセットする操作であるため、経過選択も同時にリセットする方が一貫性がある。
- **主訴別動的タブから入っても経過選択は保持される。** 経過選択はプロブレムリストパネル全体に対する状態であり、タブ切り替え（Phase 5b3〜5c相当の既存の「選択状態を消さない」設計方針）と同様の扱いとする。
- **救急モードON時も、経過選択の状態に関わらず救急候補・救急初期安定化パネルの挙動は変更しない。** 慢性を選んでいても救急モードの機能・表示には一切影響させない。

## 8. 実装する場合のテスト方針案（ONSET_DURATION_AXIS_CASES）

実装フェーズに進む場合、以下の観点を含む静的テストを追加する。

- **ODA-1**：経過選択UI（急性・亜急性・慢性・反復性・不明）が表示される
- **ODA-2**：経過選択が選択中所見エリア（または専用表示エリア）に反映される（例：急性を選ぶと「経過：急性」が表示される）
- **ODA-3**：経過未選択でも候補疾患・派生プロブレムの表示件数が変化しない（未評価として扱われることの確認）
- **ODA-4**：plResetで経過選択が解除される
- **ODA-5**：慢性を選んでも尿道閉塞・気胸・アナフィラキシー等のcritical/danger候補が除外されない（該当所見を選択した状態で経過を切り替え、候補疾患カードの存在を確認）
- **ODA-6**：pl-chip総数504件・data-fidユニーク数504件を維持し、DATA/RAW_FINDINGS/FINDINGS/DERIVED_PROBLEMS/治療本文/薬剤用量の文字列が変更前と完全一致することを確認する

既存テスト（Phase 7時点で599件相当＝非UIテスト598件＋UI遷移1件）は全件維持し、上記を追加する形とする。

## 9. 未決事項・次工程での検討事項

- 経過選択の表示場所を「選択中の所見エリア内にチップとして混在させる」か「独立した専用表示エリアにする」かは、実装時にfield feedbackも踏まえて決定する。
- 咳（`hx_chronic_cough`）のようにレベル2的な連動が部分的に可能な主訴について、期間定義の不一致（UI上の「慢性：2〜3週間以上」と既存Raw所見の「2ヶ月以上」）をどう扱うかは、実装する場合は別途整理が必要（UI上の経過選択はあくまで参考情報であり、既存Raw所見の期間定義を変更しないことを明記する等の対応が考えられる）。
- レベル3（疾患カードへのonset/durationメタデータ付与）を将来的に検討する場合は、Phase 8以降の医学的再監査プロセスの中で、主訴別必要度が「高」と評価された6主訴（嘔吐・下痢・体重減少・発作・ふらつき等・跛行）から優先的に着手することが考えられる。
