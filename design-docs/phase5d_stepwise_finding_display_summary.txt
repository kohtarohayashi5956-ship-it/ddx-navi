# phase5d_stepwise_finding_display_summary.txt
## 主訴別DDxナビ Phase 5d：検査前/身体検査後/検査後の段階表示（実装）

対象バージョン: ddx-7.3-phase5c-complaint-priority-display → ddx-7.4-phase5d-stepwise-finding-display

---

## 1. 変更概要

- Raw所見ピッカーの入力導線を、診察の進行に沿った4段階（Step1：来院時に選ぶ／Step2：身体検査後に選ぶ／Step3：検査後に選ぶ／Step4：詳細・専門所見）に再構成した。
- **Step1（来院時に選ぶ）**：Phase 5b2のENTRY_RAW_IDS（44件）の実体チップをそのまま採用し、常時展開（初期状態で開いている）で表示する。
- **Step2（身体検査後に選ぶ）**：Step1・関連する追加所見のいずれにも未使用の残りの「症状・病歴」所見（155件）を格納した。触診・聴診・神経学的検査等の身体検査所見が中心。初期状態は折りたたみ。
- **Step3（検査後に選ぶ）**：血液一般・塗抹／生化学／電解質・血液ガス・POC／尿検査／X線／超音波／心エコーの最優先・標準所見（134件）を格納した。初期状態は折りたたみ。
- **Step4（詳細・専門所見）**：体液・細胞診／特殊検査／凝固・止血／感染症・分子検査／CTの所見と、Step3側カテゴリの拡張所見（血液一般・塗抹・電解質の拡張タグ）を格納した（82件）。初期状態は折りたたみ。CTはPhase 5dの要件書の例示（「CT/MRI所見」がStep4の例）に合わせ、旧Level2（検査所見）からStep4へ移設した。
- **既存タブ・主訴別動的タブとの関係**：「よく使う」タブの実体チップ（Phase 5b4で新設したtabCommonChips）を廃止し、実体をStep1に一本化した。「よく使う」タブは、他タブ（救急・消化器等）と同じ「既存チップを参照するボタン一覧」方式に統一した。主訴別動的タブ（Phase 5c）・所見カテゴリタブ（Phase 5b3/5b4）の挙動・診断ロジックへの非影響方針は変更していない。
- **選択中の所見／直接検索／関連する追加所見**：いずれもPhase 5b3/5b2の実装をそのまま維持した（後述「4. 既存機能の維持」参照）。
- **Raw/card/DP/診断ロジック変更の有無**：**なし**。DATA・疾患カード252件・RAW_FINDINGS473件・FINDINGS31件・DERIVED_PROBLEMS143件・trigger・candidate role・treatment detail・drug doseはすべて無変更。主訴別COMPLAINT_RAW_MAPも変更していない。変更は「所見入力UIの段階表示」に限定した。

## 2. Step分類

| 区分 | 件数 |
|---|---|
| 関連する追加所見（ctx-group、Phase 5b2から維持） | 89件 |
| Step1：来院時に選ぶ | 44件 |
| Step2：身体検査後に選ぶ | 155件 |
| Step3：検査後に選ぶ | 134件 |
| Step4：詳細・専門所見 | 82件 |
| **合計** | **504件** |

- 504件（FINDINGS31件＋RAW_FINDINGS473件）との整合性：**一致**。重複・欠落はない（`data-fidユニーク数504件`で確認済み）。
- 一部の所見（例：好中球増多・BUN上昇・Cre上昇・高K・低血糖・ビリルビン上昇・腸管壁肥厚・腎盂拡張等）は、Phase 5b2から維持している「関連する追加所見（ctx-group）」に既に割り当て済みのため、Step3の物理的な内側ではなく関連追加所見側にある。これらはStep3を開かなくても、対応する入口所見（発熱・嘔吐・発作・排尿困難等）を選択すれば「関連する追加所見」から選択可能であり、機能的にはStep3相当の役割を果たす（詳細は「6. 保留・次工程」参照）。

## 4. 既存機能の維持

- **選択中の所見**：維持。どの場所（Step1〜4／関連追加所見／タブ／主訴別動的タブ／直接検索）から選んだ所見でも常時表示し、×で解除、plResetで全解除される。Step開閉・タブ切り替えでチェック状態は変化しない。
- **直接検索**：維持。FINDINGS31件＋RAW_FINDINGS473件＝504件を対象に、label／ID／catで検索し最大20件を表示、選択済みバッジも維持。
- **所見カテゴリタブ**：維持。よく使う／救急／消化器／呼吸器／泌尿器／神経／皮膚・耳・眼／血液・体腔液／内分泌／生殖器／中毒の11タブ。「よく使う」タブの実体は今回Step1へ移設したが、タブ自体の機能・診断ロジックへの非影響方針は変更していない。
- **主訴別動的タブ**：維持。各主訴カードの「🔎 プロブレムリストで所見を選ぶ」ボタン、パネル自動展開、主訴名を冠した動的タブ、該当セクションへのスクロール、動的タブからの既存Raw/FINDINGS選択、いずれも変更なし。
- **関連する追加所見**：維持。Phase 5b2のCONTEXTUAL_RAW_MAPをそのまま使用し、表示位置・仕様（選択した入口所見に応じて表示、重複なし）も変更していない。
- **派生プロブレム・整合する候補疾患**：計算ロジック・表示ロジックともに変更なし。
- **UI遷移（疾患カードを開くとパネルが閉じてカードへ移動）**：変更なし、PASSを確認済み。

## 6. 保留・次工程

- **Phase 5eへ進めるか**：進めると判断する。今回の変更もRaw所見ピッカーのUI表示（段階分け）に限定されており、DATA・疾患カード・RAW_FINDINGS・FINDINGS・DERIVED_PROBLEMS・診断ロジック・治療欄・COMPLAINT_RAW_MAPはすべて無変更、既存476件（非UI）＋UI遷移1件のテストが全てPASSしている。
- **Step分類で調整余地がある項目**：
  - 好中球増多・BUN上昇・Cre上昇・高K・低血糖・ビリルビン上昇・腸管壁肥厚・腎盂拡張など、臨床的に「検査後に選ぶ」所見の代表例が、Phase 5b2から維持している「関連する追加所見（ctx-group）」に先に割り当てられているため、Step3の物理的な内側には現れない。今回は「関連する追加所見・主訴別動的タブを維持する」という要件を優先し、既存の割り当てをそのままにした。実運用のフィードバックを踏まえ、これらの所見をStep3側にも参照リンクとして重複表示する（実体は1つを維持したまま）等の改善を次工程で検討する余地がある。
- **実運用で見直すべき表示順**：現在の推奨順（フィルター→検索→選択中→タブ→タブの中身→関連追加所見→Step1〜4→派生プロブレム→候補疾患）は要件書の推奨順に従った。実際の診察フローでの使い勝手（特にStep2「身体検査後」の155件という件数の多さ）は、実運用のフィードバックを踏まえて調整する余地がある。
- **Step1が多すぎる場合の調整案**：現在44件で要件の目安（Phase 5b2のENTRY_RAW_IDS）通りだが、さらに絞り込む場合は主訴別動的タブとの役割分担（Step1は汎用的な主訴、主訴別動的タブは主訴特異的な深掘り）を明確化し、Step1を「本当に多くの主訴で共通して使う所見」のみに絞る再選定を次工程で検討できる。
- **Step3/Step4が探しにくい場合の調整案**：Step3（134件）・Step4（82件）は件数が多いため、カテゴリ単位の折りたたみ（既存の`.raw-cat`構造）は維持しているが、検査カテゴリ内でのタグ検索・絞り込み機能の追加を次工程で検討する余地がある。
