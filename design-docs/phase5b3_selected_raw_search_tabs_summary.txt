# phase5b3_selected_raw_search_tabs_summary.txt
## 主訴別DDxナビ Phase 5b3：選択中所見の常時表示・直接検索・所見カテゴリタブ（設計/実装）

対象バージョン: ddx-7.2b-phase5b2-contextual-raw → ddx-7.2c-phase5b3-selected-raw-search-tabs

---

## 1. 変更概要

- **選択中所見エリアの実装内容**：プロブレムリスト検索パネル内に「選択中の所見」エリアを新設した。チェック済みのRaw/FINDINGSを、入口所見・関連追加所見・検査所見・詳細所見・検索・タブのどこから選んだかにかかわらず、常時1か所に集約して表示する。各チップに解除用の「×」ボタンを付け、押すと該当所見のチェックが外れる。0件のときは「所見を選択してください」の薄い表示にする。
- **直接検索欄の実装内容**：「所見名で直接検索して追加」の入力欄を新設。FINDINGS31件＋RAW_FINDINGS473件＝504件を対象に、label・ID・catの部分一致で検索し、最大20件を表示する。クリックで即チェックし、チェック済みのものには「選択済み」バッジを表示する。
- **所見カテゴリタブの実装/設計内容**：「よく使う」「救急」「消化器」「呼吸器」「泌尿器」「神経」「皮膚・耳・眼」「血液・体腔液」「内分泌」「生殖器」「中毒」の11タブを実装した（設計メモのみでなく最小実装まで行った）。タブは診断ロジックには関与せず、504件を探しやすくする「所見入力パレット」としてのみ機能する。
- **Raw/card/DP/診断ロジック変更の有無**：**なし**。DATA・疾患カード252件・RAW_FINDINGS473件のID/label/cat/priority・FINDINGS31件のID/label・DERIVED_PROBLEMS143件・trigger・candidate role・treatment detail・drug doseはすべて無変更。変更は「Raw所見ピッカーのUI/UX、選択状態の見せ方、直接検索、タブ表示」に限定した。

## 2. 選択中所見の仕様

- **入口所見を外した場合の挙動**：入口所見のチェックを外しても、その入口所見に紐づく関連追加所見のチェック状態は自動解除しない（下位所見の自動解除は行わない）。関連追加所見欄（`.ctx-group`）は非表示になるが、チェック状態は内部的に維持される。
- **チェック済み追加所見の表示維持**：関連追加所見欄が非表示になっているチェック済み所見は、「選択中の所見」エリアには必ず残る。該当チップには薄いスタイル（`.sel-chip-hidden`）と「深掘り欄未表示」の小さなフラグを付け、現在の深掘り欄には出ていないことが分かるようにした。入口所見を再度選択すると、関連追加所見欄にもチェック済み状態で再表示される（内部のチェックボックス状態は変更していないため）。
- **×解除の仕様**：「選択中の所見」の各チップの×をクリックすると、対応する実際のチェックボックス（Raw所見ピッカー内の本体）のcheckedをfalseにして`change`イベントを発火する。これにより派生プロブレム・候補疾患も通常の解除と同じ経路で再計算される。
- **plResetの仕様**：`plReset`ボタンは既存どおり、Raw所見ピッカー内のすべてのチェックボックスを解除し、`renderAll()`を再実行する。「選択中の所見」もこの再実行により自動的に空表示に戻る（個別の追加処理は不要）。

## 3. 直接検索の仕様

- **検索対象**：FINDINGS 31件＋RAW_FINDINGS 473件＝合計504件（`ALL_ITEMS_FOR_SEARCH`として起動時に一度だけ構築）。
- **検索可能項目**：label（表示名）、ID、cat（カテゴリ名）の部分一致（大文字小文字を区別しない）。例：「白血球」「好中球」「高K」「腹水」「膀胱」「しぶり」「CaOx」「胆管」等で検索可能。
- **検索結果表示数**：最大20件（`.slice(0, 20)`）。
- **選択済み所見の扱い**：検索結果の各項目は、現在チェック済みであれば「選択済み」バッジを付けて表示する。クリックすると該当所見をチェックする（新規のpl-chipは作らず、Raw所見ピッカー内の既存チェックボックスを直接操作するため、pl-chip総数・data-fidユニーク数は変化しない）。入口所見・関連追加所見・検査所見のいずれにも表示されていない所見（詳細・専門所見内や「症状・病歴・その他の所見」内のものを含む）も、検索から直接選択できる。

## 4. 所見カテゴリタブの仕様

- **タブ一覧**：よく使う（既定表示）／救急／消化器／呼吸器／泌尿器／神経／皮膚・耳・眼／血液・体腔液／内分泌／生殖器／中毒　の11タブ。
- **各タブに含める所見の考え方**：
  - 「よく使う」はPhase 5b2のENTRY_RAW_IDS（44件）をそのまま列挙。
  - 「救急」は危険所見・救急トリアージに関連する既存ID（虚脱・チアノーゼ・粘膜蒼白・発作重積・乏尿無尿・高K・GDV疑い・気胸等）を手動で列挙。
  - その他9タブ（消化器〜中毒）はlabelの部分一致キーワード（例：消化器＝「嘔吐」「下痢」「腹痛」「腸」「胃」「肝」「胆」「膵」等）で504件から緩やかに集める方式とし、最大30件を表示する。RAW_FINDINGS自体にカテゴリ属性の追加は行っていない（既存データは無変更）。
- **タブ切り替え時の選択状態保持**：タブは表示するリストの内容を切り替えるだけで、チェックボックス自体（Raw所見ピッカー内の本体）には一切触れないため、タブを切り替えても選択状態は自動的に保持される（実装上、意図的な保持ロジックは不要で、構造的に保持される）。
- **診断候補計算には影響しないこと**：タブの選択（`activeRawTab`）はUIの表示制御にのみ使用し、`computeDerivedProblems`や候補疾患フィルタのロジックには一切渡していない。タブを切り替えても派生プロブレム・候補疾患の計算結果は変化しない。

## 6. 保留・次工程

- **Phase 5cへ進めるか**：進めると判断する。今回の変更もRaw所見ピッカーのUI表示・操作性に限定されており、DATA・疾患カード・RAW_FINDINGS・FINDINGS・DERIVED_PROBLEMS・診断ロジック・治療欄はすべて無変更、既存408件＋新規17件のテストが全てPASSしている。
- **タブ分類の調整余地**：
  - 消化器〜中毒の9タブはキーワード部分一致方式のため、意図せず他領域の所見が混入する可能性がある（例：「腸」というキーワードは消化器以外の文脈でも一致しうる）。実運用のフィードバックを踏まえ、キーワードの精緻化や除外ルールの追加を検討する余地がある。
  - 現在タブは1つの所見が複数タブに重複して表示されることを許容している（タブ自体は表示専用で、Raw所見の「所属」を固定するものではないため）。これは意図した設計だが、UI上の説明を充実させる余地がある。
- **直接検索の使い勝手改善余地**：
  - 現在はlabel・ID・catの単純な部分一致のみ。表記ゆれ（例：「シュウ酸カルシウム」と「CaOx」、「白血球増多」と「好中球増多」等の同義語）への対応、ひらがな/カタカナ正規化、部位別の絞り込み等は次工程で検討する余地がある。
  - 検索結果の表示順は`ALL_ITEMS_FOR_SEARCH`内の元の並び順（FINDINGS→RAW_FINDINGS登場順）に依存しており、関連度に基づく並び替えは行っていない。
- **上位/下位所見の自動解除を今後検討する必要があるか**：今回明示的に「自動解除しない」方針を採用した（検査所見・身体所見は入口症状と独立した臨床情報であるため）。この方針は妥当と考えるが、「選択中の所見」が多くなった場合の視認性（長大なチップリストの折り返し・グルーピング表示等）は、実運用のフィードバックを踏まえてPhase 5c以降で改善を検討する余地がある。
