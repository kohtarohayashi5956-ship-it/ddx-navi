# phase5e_treatment_safety_ui_summary.txt
## 主訴別DDxナビ Phase 5e：治療欄の高リスク用量・保留項目UI改善（実装）

対象バージョン: ddx-7.4a-phase5d-step-reference-fix → ddx-7.5-phase5e-treatment-safety-ui

---

## 1. 変更概要

- 治療欄の表示関数`txHtml()`を拡張し、以下を実装した。DATA本体（`detail.tx`本文・薬剤名・用量・注記の文字列）は一切変更せず、**表示時にキーワード検出でバッジを付与し、高リスク薬剤の用量のみ折りたたみで初期非表示にする**という表示レイヤーのみの変更に限定した。
- **高リスク用量の表示方法**：薬剤名・用量・注記のいずれかに高リスクキーワード（エピネフリン、4-メチルピラゾール、NAC/N-アセチルシステイン、インスリン、カリウム、ビスホスホネート、抗血小板/抗血栓薬、化学療法/分子標的、免疫抑制薬、解毒薬等）を含む、または既存の監査フラグ`st==='caution'`を持つ薬剤を「高リスク」と判定し、カード内に「⚠️ 高リスク薬剤・用量」という折りたたみ枠を新設した。初期表示では薬剤名のみをピル表示し、「詳細を開く」ボタンを押すと既存の用量・注記文がそのまま展開表示される。
- **バッジ表示の種類**：要確認／専門判断／文献差あり／成書確認／院内プロトコル確認／感染除外／併用注意／モニタリング／避ける の9種類を実装した（詳細は「3. バッジ一覧」参照）。
- **構造整理**：既存の`tx.sum`／`tx.drugs`／`tx.esc`／`tx.monitor`／`tx.avoid`の5フィールドに対し、見出しを「🩺 まず行うこと」「💊 薬剤/処置」「⚡ 悪化時の対応／紹介・専門判断」「📋 モニタリング」「🚫 避けるべき治療」として整理した（本文の内容・並び順は変更せず、見出しラベルとバッジのみ追加）。
- **保留項目の集約表示**：各カードの治療欄先頭に「この治療欄で要注意：」という集約行を追加し、そのカード内で検出された全バッジ（高リスク用量の有無を含む）を一覧表示するようにした。
- **Raw/card/DP/診断ロジック/治療本文変更の有無**：**なし**。DATA・疾患カード252件・RAW_FINDINGS473件・FINDINGS31件・DERIVED_PROBLEMS143件・trigger・candidate role・treatment detail本文・薬剤名・薬剤用量・COMPLAINT_RAW_MAPはすべて無変更（`DATA`定数の文字列が変更前後で完全一致することを確認済み）。変更は`txHtml()`表示関数と付随CSS・クリックハンドラのみ。

## 2. 高リスク用量UI

- **対象薬剤（キーワード検出方式）**：エピネフリン、4-メチルピラゾール／4-MP／ホメピゾール、N-アセチルシステイン／NAC、インスリン、カリウム／リン酸カリウム、ビスホスホネート、抗血小板／抗血栓／クロピドグレル／ヘパリン、化学療法／分子標的／トセラニブ／ビンクリスチン／ドキソルビシン／ロムスチン、免疫抑制／シクロスポリン／アザチオプリン、解毒、抗血清、高張（液）。加えて既存の監査フラグ`st==='caution'`（6件）も高リスクとして扱う。
- **対象カード**：全252疾患カード中、いずれかの薬剤が高リスク判定された疾患カードは**65件**（対象薬剤エントリ数は614件中80件）。重点対象カード（collapse_anaphylaxis、vomit_ethylene_glycol_toxicosis、jaundice_acetaminophen_toxicosis、vomit_diabetic_ketoacidosis、pupd_electrolyte_disorders、dysuria_urethral_obstruction等）はすべて高リスクUIの対象として検出されることを確認済み。
- **表示方法**：カード内の「💊 薬剤/処置」の直下に「⚠️ 高リスク薬剤・用量（n件）」という枠を新設し、対象薬剤名をピル表示。「詳細を開く」ボタンで用量・注記を展開する（再度押すと「閉じる」で折りたたむ）。
- **折りたたみの有無**：**あり**。初期状態は折りたたみ（非表示）。ボタンクリックで開閉をトグルする。
- **用量文を変更していないこと**：既存の用量表示ロジック（`g.st`による「要確認」表示等）はそのまま再利用し、折りたたみ内のマークアップは変更前の`<li>`と同じ構造・同じテキストを使用している。DATA定数の文字列比較により、治療本文・用量は1文字も変更されていないことを確認済み。

## 3. バッジ一覧

| バッジ | 検出キーワード（正規表現） | 表示される文脈 |
|---|---|---|
| 要確認 | `要確認`／`確認が必要`／`確認する(こと)?` | sum／esc／avoid／薬剤note内にこれらの語がある場合。また用量が`st`未確定（'ok'/'std'以外）で「要確認」表示されている薬剤にも個別付与 |
| 専門判断 | `専門(医\|科\|的)`／`紹介`／`転院` | 専門医・専門科への相談、転院を示す記述がある場合（例：mass_mast_cell_tumor「専門的」、bleeding_disseminated_intravascular_coagulation「専門的判断」、collapse_feline_aortic_thromboembolism「専門医/循環器科と相談」） |
| 文献差あり | `文献(間\|差\|照合)` | 文献間で用量・プロトコルに差があることを示す記述（例：collapse_anaphylaxis「文献間で高用量プロトコルの記載もあり」、jaundice_acetaminophen_toxicosis「投与間隔は文献間で4〜6時間の幅がある」） |
| 成書確認 | `成書` | 成書（教科書）での確認を促す記述 |
| 院内プロトコル確認 | `院内プロトコル` | 院内プロトコルでの調整・確認を促す記述（例：collapse_anaphylaxis「重症度に応じて院内プロトコルで調整する」） |
| 感染除外 | `感染.{0,12}除外`／`除外.{0,12}感染` | ステロイド等の免疫抑制導入前に感染性疾患の除外を促す記述（例：lameness_immune-mediated_polyarthritis「感染性関節炎を除外しないままの免疫抑制薬導入」） |
| 併用注意 | `併用` | 薬剤の併用に関する注意記述（例：vomit_gastritis_gastric_ulcer、lameness_soft_tissue_traumaのNSAIDs/ステロイド併用回避） |
| モニタリング | （`tx.monitor`フィールドが存在する場合は常時付与） | モニタリング項目が記載されているすべてのカード |
| 避ける | （`tx.avoid`フィールドが存在する場合は常時付与） | 避けるべき治療が記載されているすべてのカード |

## 8. Phase 5fへ進める状態か

**進めると判断する。** 今回の変更は治療欄の表示構造・視認性・注意喚起UIに限定されており、DATA・疾患カード・RAW_FINDINGS・FINDINGS・DERIVED_PROBLEMS・診断ロジック・治療本文・薬剤用量・COMPLAINT_RAW_MAPはすべて無変更（文字列完全一致で確認済み）、既存491件（非UI）＋UI遷移1件＋新規30件のテストが全てPASSしている。

次に扱うべきUI課題（Phase 5f以降での検討候補）：

- **キーワード検出の精緻化**：現在のバッジ・高リスク判定はキーワードの部分一致に依存している。誤検出（例："併用"が別の文脈で使われている等）や検出漏れ（例：中毒相談窓口の連絡先、蛇咬傷の抗血清等、まだキーワード対象に含めていない保留項目）を実運用のフィードバックを踏まえて調整する余地がある。
- **高リスク薬剤リストの拡充**：現在は約25キーワードで判定している。降圧薬・鎮静薬・麻酔薬等、他の高リスク領域への拡充を検討できる。
- **バッジの視覚的優先度**：現在すべてのバッジが同じサイズ・配置で並ぶ。「⚠️ 高リスク用量あり」等、最重要な警告を視覚的に強調する（色・アイコンサイズ等）改善の余地がある。
- **カードリスト全体での高リスクカードの一覧性**：現在は各カード内でのみ高リスクUIが見える。主訴一覧やプロブレムリストの候補疾患表示（`.cand-card`）側にも「⚠️」等の簡易インジケータを追加し、開く前に気づけるようにする改善を検討できる。
